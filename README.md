# Веб-платформа записи на занятия и управления расписанием

## 📌 Назначение
Платформа предназначена для автоматизации процесса записи на занятия, управления расписанием преподавателя, отправки напоминаний и уведомлений, обработки оплат и ведения учёта.  
Обеспечивает удобные интерфейсы для учеников и преподавателей.

---

## ⚙️ Технологии

### Backend
- [Django REST Framework] — текущая реализация API в этом репозитории

### Frontend
- React + Vite — SPA-интерфейс

---

## Запуск локально

### Docker Compose

```bash
# из корня репозитория
docker compose build
docker compose up -d

# фронтенд: http://localhost:5173
# API: http://localhost:8000/api/v1/
# Django Admin: http://localhost:8000/admin/ (admin/admin)
```

**Автоматически создается суперпользователь:**
- Логин: `admin`
- Пароль: `admin`
- Доступ к Django Admin: http://localhost:8000/admin/

Остановить:
```bash
docker compose down
```


Фронт будет на `http://localhost:5173`. Запросы к `/api` проксируются на `http://localhost:8000` (см. `frontend/nginx.conf` для Docker и `frontend/vite.config.ts` для dev-режима).

##  Что ещё надо доделать в бэкенде
- Аутентификация и роли (student/teacher) — сейчас нет логина и токенов
- Валидация времени (проверка пересечений слотов и форматов дат)
- Связь с реальным `User` при создании слота (сейчас требуется `teacher` как ID)
- Пагинация/фильтрация списка сессий на API
- Статусы и правила переходов (available → scheduled → completed)
- CORS-политика на прод (в dev закрыто прокси)

## 🗂 Декомпозиция системы

### Микросервисы
1. **Auth Service** — регистрация, авторизация, управление ролями  
2. **User Service** — профили пользователей, настройки уведомлений  
3. **Schedule Service** — календарь занятий, бронирования, переносы, отмены  
4. **Notification Service** — email/SMS-уведомления, интеграция с Celery  
5. **Payment Service** — обработка оплат и интеграция с провайдерами  
6. **Integration Service** — синхронизация с Google Calendar  
7. **Frontend** — веб-интерфейс для студентов и преподавателей  
8. **Admin Panel** — управление системой (для преподавателей/администратора)

## ✅ План реализации

### Этап 1. Подготовка (высший приоритет)
- Настройка репозитория и CI/CD
- Развёртывание инфраструктуры (Docker Compose)
- Проектирование схемы БД

### Этап 2. Основная функциональность
1. **Auth Service** — регистрация/логин, роли (**критично**)  
2. **User Service** — профили и личные данные (**критично**)  
3. **Schedule Service** — календарь и бронирования (**критично**)  

### Этап 3. Вспомогательная функциональность
4. **Notification Service** — напоминания по email/SMS (**высокий приоритет**)  
5. **Payment Service** — онлайн-оплата (**высокий приоритет**)  
6. **Integration Service** — синхронизация с Google Calendar (**средний приоритет**)  

### Этап 4. Интерфейс
7. **Frontend (React)** — веб-клиент (**средний приоритет**)  
8. **Admin Panel** — панель управления (**низкий приоритет**)  

### Этап 5. Нефункциональные задачи
- Написание тестов (unit, интеграционные, нагрузочные)  
- Подготовка документации (API, руководство пользователя, гайд по развёртыванию)  
- Настройка мониторинга и логирования
