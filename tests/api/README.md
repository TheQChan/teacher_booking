# API-тесты расписаний и регистрации

В `tests/api/` живёт тот же набор Django `TestCase`. Они проверяют:
- разрешения на чтение/создание слотов,
- бизнес-потоки записи/освобождения/отмены/завершения,
- регистрационный сценарий и endpoint `/api/v1/drf-auth/user/`

### Запуск
1. Убедитесь, что зависимости установлены:  
   `python -m pip install -r ../requirements.txt` 
2. Выполните `pytest api/test_session_api.py`
