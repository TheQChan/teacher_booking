# Тесты Teacher Booking

Каталог `teacher_booking/tests/` содержит вынесенный набор проверок backend-а — самописный `pytest`‑/Django‑раннер, план тестирования и описания кейсов. Разделение по папкам позволяет добавлять новые типы (BDD, GUI/selenium, нагрузка) без засорения основного API‑набора.

## Структура
- `api/` — текущие Django `TestCase` с проверками API расписаний, регистрации и доступа.
- `bdd/` — сюда позже добавим `feature`‑файлы и шаги для поведенческих сценариев (например, на `behave` или `pytest-bdd`).
- `gui/` — сюда будут вынесены Selenium/Playwright‑тесты интерфейса, которыми можно управлять отдельно от API.
- `load/` — папка для нагрузочных/спецификаций по производительности (например, `locust`, `k6` или `JMeter`).
- `test_cases/` — человекочитаемые сценарии, соответствующие каждому тесту из `api/`.
- `requirements.txt` — зависимости Django + `pytest`/`pytest-django`, можно расширить `bdd`/`gui`/`load` пакетами.
- `conftest.py` — точка входа, которая подключает соседний backend (`../backend/teacher_booking`) и импортирует его настройки.

## Подготовка
1. Убедитесь, что каталог `teacher_booking/tests/` находится рядом с `backend/` и `frontend/`, потому что `conftest.py` ищет бекенд по пути `../backend/teacher_booking`.
2. Установите зависимости из `requirements.txt` и, при необходимости, дополняйте их пакетами из `bdd/`, `gui/`, `load/`:
   ```bash
   python -m pip install -r requirements.txt
   ```

## Запуск существующих API-тестов
```bash
pytest
```
`pytest` автоматически использует `conftest.py`, загружает настройки Django и выполняет всё из `api/`.

## Расширение
- Новые API‐тесты добавляйте под `api/`, синхронизируя описание в `test_cases/`.
- Для BDD-компонента создайте `feature`‑файлы и соответствующие шаги в `bdd/`, указывая в README внутри папки, как их запускать.
- GUI/selenium сценарии живут в `gui/`; можно добавить свою `conftest`‑конфигурацию, если потребуется отдельный драйвер.
- Нагрузочные проверки помещайте в `load/` (например: `load/locustfile.py`).
- Всегда фиксируйте зависимости и инструкции запуска в `README.md` или внутри соответствующего подпапки, чтобы поддерживать ясность.
