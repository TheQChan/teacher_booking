# Тест-кейсы API расписаний

## Разрешения и просмотр

### Анонимный пользователь не может получить список сессий
- **Шаги**: выполнить `GET /api/v1/session/` без авторизации.
- **Ожидается**: HTTP 403, потому что неавторизованные пользователи не должны видеть расписание.

### Авторизованный студент может просматривать список
- **Шаги**: авторизоваться как студент и повторить вызов `GET /api/v1/session/`.
- **Ожидается**: HTTP 200 с массивом слотов (даже если он пуст), потому что любой залогиненный пользователь может читать доступные интервалы.

### Студент не может создать слот
- **Шаги**: авторизоваться как студент и `POST /api/v1/session/` с валидными `start_time`/`end_time`.
- **Ожидается**: HTTP 403 — только учитель может публиковать своё расписание.

### Учитель может создать слот
- **Шаги**: авторизоваться как учитель и выполнить тот же `POST`.
- **Ожидается**: HTTP 201, тело возвращает созданный слот с `teacher` равным ID текущего пользователя.

## Бизнес-правила

### Наложение слотов отклоняется
- **Шаги**: учитель создаёт слот, затем пытается создать второй, который перекрывается на 30 минут.
- **Ожидается**: второй вызов возвращает HTTP 400 и текст ошибки упоминает наложение/конфликт по времени.

### Проверка окончания раньше начала
- **Шаги**: учитель создаёт слот, где `end_time` раньше `start_time`.
- **Ожидается**: HTTP 400 с валидационной ошибкой.

### Полный поток schedule/free/cancel/complete с проверками
- **Шаги**:
  1. Учитель создаёт слот.
  2. Студент назначает его (`POST /schedule/`).
  3. Тот же студент снова пытается назначить (должна быть ошибка, что слот уже занят).
  4. Другой студент пытается освободить слот (ожидается отказ).
  5. Первоначальный студент освобождает слот.
  6. Повторный `free` возвращает HTTP 200, но с текстом о том, что слот уже свободен.
  7. Студент снова назначает слот.
  8. Не-учитель пытается отменить слот (должна быть проверка прав).
  9. Учитель отменяет слот успешно.
  10. Попытка завершить слот, который не в состоянии «запланирован», возвращает ошибку.
- **Ожидается**: каждый шаг возвращает ожидаемый код (обычно 200, кроме проверки ошибок) и понятный текст ошибки, если действие запрещено.
