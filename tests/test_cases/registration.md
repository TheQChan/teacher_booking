# Тест-кейсы регистрации и /me

## Подтверждение пароля и уникальность
- **Шаги**:
  1. Отправить `POST /api/v1/register/` с разными `password` и `password_confirm`.
  2. Отправить `POST /api/v1/register/` с уникальным логином/почтой и совпадающими паролями.
  3. Повторить `POST`, использовав email из шага 2.
- **Ожидается**:
  1. HTTP 400 — пароли не совпадают.
  2. HTTP 201, в ответе указан телефон, роль «teacher» и вложенный объект `user`.
  3. HTTP 400 — email должен быть уникальным.

## Текущий пользователь
- **Шаги**: авторизоваться как заранее созданный студент и выполнить `GET /api/v1/drf-auth/user/`.
- **Ожидается**: HTTP 200 с полями `username` и `role`, совпадающими с текущим пользователем.
